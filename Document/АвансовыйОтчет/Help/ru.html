Документ "Авансовый отчет"

Общее

Документ "Авансовый отчет" предназначен для ведения расчетов с подотчетными лицами.

Форма шапки документа

При добавлении нового документа открывается окно "Авансовый отчет" .

Номер, дата и автор указываются в шапке документа автоматически.

В шапке документа нужно заполнить сдедующие поля:

        • Фирма, от лица которой ведутся взаиморасчеты. Элемент справочника "Фирмы".
        • Сотрудник, с которым ведутся взаиморасчеты. Элемент справочника "Сотрудники".


Документ  "Авансовый отчет" отображается на печати с двух сторон и содержит вкладки:

        • Лицевая сторона
        • Оборотная сторона

Вкладка "Лицевая сторона".

На данной вкладке необходимо заполнить следующие реквизиты:

        • Предыдущий аванс. Сумма предыдущего аванса (указывается, был ли это остаток выданного ранее аванса либо перерасход по этому авансу).
        • Валюта. Заполняется из выпадающего списка.
        • Место выдачи аванса. Например, Магазин №1, Центральный офис. Элемент справочника "Структурные единицы".
        • Назначение аванса. Заполняется вручную.

В табличной части необходимо заполнить следующие колонки:

        • Расходный кассовый ордер. Документ, по которому был выдан аванс. Элемент журнала расходных кассовых ордеров.
        • Сумма выданного аванса.

В нижней части окна заполняются поля:

            • Количество приложенных документов. Заполняется вручную.
            • Количество приложенных листов. Заполняется вручную.

Вкладка "Оборотная сторона"

На данной вкладке необходимо заполнить следующие реквизиты:

            • Классификация операции/Корр. счет. Классификация операции заполняется из выпадающего списка: расчеты с поставщиками или прочие выплаты и поступления. Корр. счет заполняется при помощи плана счетов бухгалтерского баланса.
            • Дата/номер/ наименование документа расхода. Дату можно выбрать с помощью календаря, номер и наименование документа указывается вручную. Информация об оправдательном документе расхода подотчетной суммы.
            • Сумма к учету/Сумма по отчету. Сумма к учету заполняется вручную. По умолчанию сумма по отчету дублирует сумму к учету. В случае несовпадения  - сумму по отчету можно изменить вручную. Сумма к учету - сумма, принимаемая бухгалтерией к учету. Сумма по отчету –  сумма затрат, заявленная сотрудником.
            • Разрезы аналитики.  Заполняется автоматически разрез аналитики для классификации операции "расчеты с поставщиками" - "Контрагент" и "Договор".
            • Значения.  Указываются значения аналитики для классификации операции "расчеты с поставщиками" -  "Контрагент" и "Договор" из  соответствующих справочников.
            • Документ обязательства. При помощи подбора необходимо выбрать документ, по которому отчитывается сотрудник - "Приходная накладная" или "Приобретение ОС, НМА, РБП".
            • Валюта обязательства/Курс валюты договора/Ставка НДС. Валюта обязательства является элементом справочника  "Валюты" (заполняется автоматически из документа-обязательства). Курс указывается автоматичеки из договора. Ставка НДС заполняется автоматически из договора, является элементом справочника "Ставки НДС".
            • Сумма вал./Сумма руб./Сумма НДС. Заполняется вручную. Сумма руб. -  рублевое покрытие обязательства на день поставки товара.
            • Статья движения денежных средств/Комментарий. Статья движения денежных средств  заполняется из справочника "Статьи движения денежных средств". Комментарий заполняется вручную.

При помощи кнопки меню "Создать операцию" система создает документ "Финансовая операция", предназначенный для финансовой аналитики.
